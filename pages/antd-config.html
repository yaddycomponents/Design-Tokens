AntD v5 · Usage
ConfigProvider Setup
Wrap your app root with ConfigProvider to apply the theme globally.
AntD v5 TSX App.tsx 
import { ConfigProvider } from 'antd';
import theme from './theme/tokens.v5';

// Font imports (npm install @fontsource/inter)
import '@fontsource/inter/300.css';
import '@fontsource/inter/400.css';
import '@fontsource/inter/500.css';
import '@fontsource/inter/600.css';
import '@fontsource/inter/700.css';

export default function App() {
  return (
    <ConfigProvider theme={theme}>
      {/* your app */}
    </ConfigProvider>
  );
}
✦ Opt-in CSS Variables in v5
To enable CSS variables mode in v5, add cssVar: true inside your theme object. This outputs --ant-* CSS custom properties onto :root.
AntD v6 · Seed Tokens
theme/tokens.v6.ts
v6 uses CSS Variables by default. The token structure is the same but the runtime is pure CSS — no cssinjs overhead.
ℹ v6 CSS Variables Mode

AntD v6 TypeScript hooks/useAIToken.ts 
import { theme } from 'antd';

// AI token primitive values
export const AI_TOKEN_VALUES = {
  pink:        '#FFF0F6',  // magenta-1
  blue:        '#F0F5FF',  // geekblue-1
  pinkLight:   '#FFF0F6',  // magenta-1
  blueLight:   '#F0F5FF',  // geekblue-1
  pinkMid:     '#FFD6E7',  // magenta-2
  blueMid:     '#D6E4FF',  // geekblue-2
} as const;

// CSS variable names for use in JS
export const AI_CSS_VARS = {
  text:        'var(--ai-theme-text)',
  background:  'var(--ai-theme-background)',
  icon:        'var(--ai-theme-icon)',
  border:      'var(--ai-theme-border)',
} as const;

// Combined hook — returns AntD token + cssVar map + AI tokens
export const useAIToken = () => {
  // v6: useToken returns { token, cssVar } where cssVar gives --ant-* names
  const { token, cssVar } = theme.useToken();
  return {
    ...token,
    antdCssVar: cssVar,
    ai:         AI_TOKEN_VALUES,
    aiCssVar:   AI_CSS_VARS,
  };
};

⚠ zeroRuntime note
Adding zeroRuntime: true to the theme config requires a build plugin (@ant-design/cssinjs-vite-plugin for Vite or @ant-design/cssinjs-webpack-plugin for webpack). It statically extracts all CSS at build time for the best possible performance.
